# Build and Run

## Stack

- Language: TypeScript and JavaScript.
- Runtime: Node.js, browser, Docker Compose for local full stack.
- Client framework: Vite, vanilla DOM, Three.js.
- Server framework: Node `http.createServer` plus `ws`, no Express.
- Database: PostgreSQL 16 for accounts, characters, market, moderation, social
  data, and sessions.
- Tests: Vitest plus script-level smoke and integration tools.

Primary files:

- `package.json`
- `vite.config.ts`
- `Dockerfile`
- `docker-compose.yml`
- `.env.example`
- `tsconfig.json`

## Common Commands

```bash
npm ci
npm run build
npm test
npm run dev

npm run server
npm run build:env
npm run env
npm run bench
```

## Local Modes

Offline browser mode:

```bash
npm run dev
```

Open the Vite site and choose offline play. Offline mode runs a local `Sim`
inside the browser.

Online dev mode with hot reload:

```bash
npm run db:up
npm run server
npm run dev
```

Full local Compose stack:

```bash
cp .env.example .env
docker compose up -d --build
```

This checkout also has a local-only `docker-compose.override.yml` ignored by Git
that maps this machine's instance to:

- Game: `http://127.0.0.1:18787/`
- Wiki: `http://127.0.0.1:18081/wiki/`
- Postgres: `127.0.0.1:15433`

## Generated and Build Outputs

- `dist/`, `dist-server/`, and `dist-env/` are build outputs.
- `src/ui/i18n.resolved.generated/` and `src/admin/i18n.resolved.generated/`
  are generated locale tables.
- `src/render/assets/manifest.generated.ts` is generated by the media manifest
  build step.
- `src/game/sfx_manifest.generated.ts` and
  `src/game/voice_manifest.generated.ts` are generated audio manifests.
- `media-cache/` and `/tmp/wocc-media-cache` are runtime media cache locations.
