# Asset Pipeline

## Purpose

Track where runtime media lives, how it is discovered, and how it reaches the
renderer, UI, audio, and guide.

## Primary Asset Directories

- `public/models/` contains glTF assets for characters, creatures, dungeon kits,
  props, foliage, resources, tools, weapons, and quest objects.
- `public/textures/` contains terrain, water, and skin textures.
- `public/env/` contains HDR and backdrop environment media.
- `public/vfx/` contains particle sprites.
- `public/ui/` contains cursor, emote, skill, and weapon UI assets.
- `public/audio/sfx/` and `public/audio/voice/` contain runtime sound assets.
- `docs/screenshots/` contains documentation and README screenshots.

## Primary Code

- `src/render/assets/loader.ts` loads glTF, HDR, and texture assets.
- `src/render/assets/preload.ts` collects boot-time preload promises.
- `src/render/assets/manifest.generated.ts` is generated by
  `scripts/build_media_manifest.mjs`.
- `src/render/props.ts`, `foliage.ts`, `dungeon.ts`, `terrain.ts`, `sky.ts`,
  `water.ts`, `quest_objects.ts`, and `vfx.ts` register render asset preloads.
- `src/game/sfx_manifest.generated.ts` and `src/game/voice_manifest.generated.ts`
  are generated audio manifests.
- `scripts/assets/`, `scripts/sfx/`, and `scripts/voices/` contain asset-related
  tooling.

## Boot Data Flow

```text
Render/audio modules imported
-> module-level preload registration
-> startGame() calls assetsReady()
-> Renderer builds scene after assets resolve
-> Renderer.prewarmInitialScene()
```

## Runtime Media Cache

Docker Compose maps `EASTBROOK_MEDIA_DIR` to `/app/dist/media` inside the game
container. On this workstation the ignored override uses `/tmp/wocc-media-cache`
with a container label option for the bind mount.

## Change Risk

Medium to high. Asset path changes can break production builds, preload gates,
runtime cache emission, and visual startup. Do not rename or move public assets
without checking render modules and generated manifests.

## Verification Steps

- Run `npm run build`.
- Start the game and enter world.
- Confirm no loading-screen asset failure.
- Check browser console for missing `public/` paths.
- For model changes, inspect both nearby world props and guide model viewer if
  relevant.
